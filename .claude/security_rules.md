[RULE 10] 작업 완료 허위 보고 금지 (가장 중요)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- 작업 완료 보고는 반드시 아래 증거 중 하나 이상 첨부:
    파일 작성: cat [파일경로] 출력 결과 일부
    git 커밋: git log --oneline -1 실제 해시값
    테스트 통과: 실제 실행 결과 출력
    설치 완료: pip show 또는 which [명령어] 결과

- 아래 상황에서 완료 보고 절대 금지:
    실제 파일이 생성되지 않았는데 "생성 완료" 보고
    오류가 발생했는데 "성공" 보고
    테스트를 실행하지 않았는데 "테스트 통과" 보고
    git push가 실패했는데 "push 완료" 보고

- 오류 발생 시 처리 규칙:
    1. 오류 메시지 전문 출력 (생략 금지)
    2. 루트 코즈 분석 (왜 실패했는가)
    3. 대안 3가지 제시
    4. 사용자 선택 후 진행
    5. 동일 오류 3회 반복 시 작업 중단 + 사용자에게 수동 조치 요청

- 토큰 절약을 위한 요약/생략 금지:
    "나머지는 동일합니다" 금지
    "위와 같은 방식으로" 금지
    실제 실행하지 않은 단계를 완료로 표시 금지

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
모든 작업 완료 후 아래 질문에 스스로 답하고 출력:

  Q1. 실제로 파일이 생성/수정됐는가? (dir/cat으로 확인)
  Q2. 실제로 테스트가 통과됐는가? (실행 결과 출력)
  Q3. 보안 취약점이 생기지 않았는가? (키 노출, 권한 과잉)
  Q4. 다음 작업에 영향을 주는 변경사항이 있는가?
  Q5. 사용자에게 수동으로 해달라고 할 항목이 있는가?

모든 Q에 Yes/No + 근거 출력. 불확실하면 "미확인 — 수동 확인 필요" 표시.

[RULE 12] 자율 판단 및 실행 규칙
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- "진행할까요?" 금지
- "그게 맞을까요?" 금지
- "확인이 필요합니다" 후 대기 금지
- "어떻게 하시겠어요?" 금지

대신:
  1. 직접 사실 확인 (dir/cat/curl/git log 등)
  2. 확인 결과 기반으로 최선안 스스로 판단
  3. 판단 근거 한 줄로 출력
  4. 즉시 실행
  5. 완료 후 결과 보고

사용자에게 물어봐야 하는 경우는 딱 2가지만:
  A. 돌이킬 수 없는 삭제 (DB 전체 삭제, git history 소각 등)
  B. 보안 정책 변경 (키 교체, 접근 권한 변경 등)
  이 2가지 외에는 스스로 판단하고 실행.

불확실한 것이 있으면:
  "불확실" 표시 후 가장 안전한 방향으로 자동 선택.
  사용자에게 질문 던지고 대기하는 것 금지.
