## 📋 **공유기 설정 (사용자 직접 실행)**

### 1️⃣ **WiFi 보안** (필수)
```
공유기 접속: http://192.168.0.1 또는 192.168.1.1

설정 항목:
1. 암호화 방식: WPA3 (또는 WPA2/AES)
2. WiFi 비밀번호:
   - 최소 20자 이상
   - 대문자/소문자/숫자/특수문자 혼합
   예) K9xL@2m#Qp7vN5rT8b!

3. 채널 설정 (DFS 문제 해결):
   ⭐ 권장: 80MHz (안정적)
     - 160MHz 대신 사용
     - DFS 자동 채널 변경 최소화
     - RDP 연결 안정성 ↑

   또는: 2.4GHz 대역
     - DFS 없음 (채널 고정)
     - 더 안정적 (권장 ⭐⭐)
```

### 2️⃣ **MAC 필터링 — "화이트리스트" 모드** (필수)
```
⭐ 중요: "화이트리스트" (명시적으로 허용된 것만)

허용할 MAC (4개):
✅ 40:B0:76:A1:EF:A0     Scarlett_Main_PC (현재)
✅ AA:BB:CC:DD:EE:01     향후_노트북
✅ BB:CC:DD:EE:FF:02     향후_iPad
✅ CC:DD:EE:FF:00:03     향후_iPhone

⛔ 모든 다른 기기: 자동 차단
```

### 3️⃣ **포트 포워딩 — 모두 비활성화** (필수)
```
현재 공개 포트:
❌ SSH (22) — 차단
❌ HTTP (80) — 차단
❌ HTTPS (443) — 필요 없음 (RDP 4389만 사용)
❌ RDP (3389) — 이미 비활성화 (4389로 변경됨)
❌ SMB (445) — 이미 차단됨
❌ 그 외 모든 포트 — 차단

결과: 포트 포워딩 규칙 0개 (완전 폐쇄)
```

### 4️⃣ **UPnP (Universal Plug and Play) — 비활성화** (필수)
```
⛔ UPnP 활성화 시 악성 앱이 자동으로 포트 개방 가능
✅ 비활성화 (앱 권한 제거)
```

### 5️⃣ **WPS (WiFi Protected Setup) — 비활성화** (필수)
```
⛔ WPS PIN 방식은 8자리만으로 2시간 안에 뚫림
✅ 비활성화 (강제)
```

### 6️⃣ **공유기 관리 인터페이스** (완료됨)
```
✅ 관리자 비밀번호: 기본값 변경 완료
✅ HTTP 관리: 비활성화 (HTTPS만)
```

## ✅ **백엔드 보안** (나가 이미 한 것)

### **RDP 원격 업무 환경**
```
포트: 4389 (커스텀, 자동 스캔 회피)
접근 가능: Render IP(115.22.193.150/32)만
암호화: FIPS 140-2 AES-256
인증:
  1. Windows 계정 비밀번호
  2. Device Registry MAC 검증
  3. IP 화이트리스트 (Render)

결과: 나만 접근 가능 ✅
```

### **로컬 네트워크 차단**
```
❌ SMB (445) 차단 — 파일 공유 불가능
✅ RDP (4389) 만 허용 — 원격 데스크톱만

결과: 공유 폴더 없음 ✅
```

## 🚀 **사용 흐름**

```
1. 공유기 MAC 필터링 → Scarlett PC만 WiFi 접근
2. Scarlett PC WiFi 연결 → 내 기기 자동 인식
3. scripts/rdp_secure_connect.bat 실행
4. MAC 검증 → Device Registry 확인
5. RDP 포트 4389 연결 → Render 경유
6. Windows 계정 로그인
7. 원격 업무 시작 ✅

⛔ 다른 누구도: WiFi 연결 불가 (MAC 필터링)
⛔ 포트 스캔: 포트 없음 (0개 포워딩)
⛔ 무차별 공격: Render IP 화이트리스트 (115.22.193.150만)
```

## ⚠️ **주의사항**

### WiFi 채널 DFS 문제
```
현재: 160MHz + DFS = 채널 자동변경 가능

문제: RDP 사용 중 WiFi 채널 변경 → 연결 끊김

해결:
✅ 채널 크기: 80MHz로 변경 (권장)
✅ 또는 2.4GHz 대역 사용 (더 안정)
❌ DFS 비활성화 금지 (레이더 간섭 위험)
```

### 다른 기기 미래 추가
```
아이패드/아이폰/노트북 추가 시:
1. 기기 MAC 주소 확인
2. Device Registry에 등록
3. 공유기 MAC 필터링에 추가
```
