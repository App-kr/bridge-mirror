SECURITY FLOW: UNAUTHORIZED ACCESS ATTEMPT
================================================================================

Scenario: Attacker sends GET /api/admin/candidates with x-admin-key: dummy

1. Request arrives at FastAPI controller
2. _check_admin(request) is called immediately
3. hmac.compare_digest("dummy", ACTUAL_KEY) → False
4. HTTPException(403) raised
5. Request stops here ← CRITICAL: Database never queried
6. Response sent: {"isError": true, "errorCategory": "ADMIN_KEY_INVALID"}
7. PII: NEVER DECRYPTED, NEVER EXPOSED ✅

Result: No PII leakage possible without valid key

SECURITY FLOW: AUTHORIZED ADMIN REQUEST
================================================================================

Scenario: Admin sends GET /api/admin/candidates with valid x-admin-key

1. Request arrives with valid key
2. _check_admin(request) → hmac.compare_digest(key, ACTUAL_KEY) → True

4. _decrypt_row() decrypts all AES-256-GCM fields

6. Valid admin detected → BYPASS masking (intentional)
7. Full unencrypted data returned to admin ✅
8. Admin needs this data to: send emails, update records, hire candidates

Result: Admin has full operational access while maintaining defense-in-depth
